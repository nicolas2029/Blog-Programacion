Introduccion

Se utiliza frecuentemente al momento de leer configuraciones y cargar una pool de conexiones a la base de datos.

El patron singlenton crea una unica instancia para utilizarla en toda la aplicacion.

Implementacion en golang

  1. Nombramos al package

  2. Importamos el package sync

  3. Creamos dos variables globales no exportadas

    • Una variable once sync.Once

    • Una variable cualquiera *type

  4. Funcion init, dentro de esta funcion llamaremos la funcion one.Do(func(){})

  5. Dentro de la funcion Do añadimos nuestra funcion anonima que se ejecutara una unica vez, en donde crearemos nuestro objeto type, type puede ser de cualquer tipo, lo que creamos lo guardamos en nuestra variable global creada previamente.

  6. Creamos una funcion para retornar nuestra variable no exportada
