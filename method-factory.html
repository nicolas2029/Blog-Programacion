Repositorio en github

Introduccion

Define una interfaz para crear un objeto, las subclases crearan los objetos.

Se utiliza para la gestion de diferentes bases de datos, implementacion de diversos metodos de pagos.

Se utiliza cuando se necesita crear un objeto, pero aun no se sabe que objeto crear.

Crea objetos concretos en tiempo de ejecucion.

El patron de diseño necesita de 4 elementos para funcionar correctamente:

  • Producto: define la interfaz de los objetos que crea el metodo de fabricacion

  • Producto Concreto: implementa la interfaz del producto

  • Creador: declara los metodos para fabricar, devuelve un objeto del tipo indicado

  • Creador Concreto: redefine el metodo de frabricacion para devolver la instancia del producto concreto

Pasos para utilizar

  • Se creara un archivo que contendra el Producto(Interfaz).

  • Se creara un archivo por cada Producto concreto(Producto que implementa la interfaz)

  • Se creara un archivo que contendra el creador y ceador concreto

    • Tendremos un enum o const que contendra los casos(identificador tipo string de cada objeto concreto)
    • Un switch con los casos previamente creador, cada caso detornara el objeto concreto indicado
